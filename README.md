# 模组灵译 Mod Lingo

> 面向玩家的 Minecraft 模组 / Addon 翻译工作区工具
> A player-first translation workspace for Minecraft mods and addons.

## 项目简介

**模组灵译（Mod Lingo）** 是一款面向 Minecraft 玩家与汉化维护者的跨平台桌面工具。

它的目标不是做一个“只能翻译单个文件”的小脚本，而是提供一个更接近 **VS Code 工作区** 的翻译体验：
用户打开一个项目，就是打开一个独立工作区；
如果需要同时处理多个项目，就分别使用多个窗口。

项目初期将聚焦于最常见、最稳定、最容易形成闭环的翻译场景：

- `lang`
- `json`
- `js`

当前阶段主要处理这些文件中的**键值型文本翻译**，优先把“导入 → 浏览 → 翻译 → 导出”这条流程打通。

---

## 项目定位

Mod Lingo 希望成为一个真正**面向玩家**的 Minecraft 翻译工具，而不是只给少数开发者使用的命令行脚本。

它优先解决的是这些问题：

- 英文模组 / Addon 游玩门槛高
- 语言文件分散，维护成本高
- 多文件翻译过程缺少统一工作区
- 玩家想批量处理文本，而不是手工改文件
- 汉化维护者需要一个更清晰的项目视图和编辑流程

---

## 当前阶段目标

项目初期先专注做好以下能力：

- 工作区管理
- 多窗口项目打开
- `lang` / `json` / `js` 文件导入
- 键值文本识别
- 键值翻译与编辑
- 翻译结果导出

当前不会过度承诺复杂能力，例如：

- 通用硬编码文本扫描
- 字节码级文本提取
- 任意脚本语义分析
- 所有 Mod / Addon 格式的全自动兼容

这些方向未来可以继续扩展，但不是第一阶段重点。

## 工作区设计

Mod Lingo 的核心交互模型参考 VS Code：

- **一个窗口 = 一个工作区**
- **一个工作区 = 一个独立项目上下文**
- **多个工作区 = 多个独立窗口**

这样设计的原因是：

- 更适合管理不同模组 / Addon 项目
- 降低多项目之间的状态污染
- 更清晰地维护项目缓存、翻译记录与导出结果
- 更符合桌面端重度工具的使用习惯

### 工作区内计划包含的内容

- 项目基础信息
- 文件树
- 待翻译键值列表
- 已翻译 / 未翻译状态
- 搜索与过滤
- 导出配置
- 项目缓存与翻译记录（后续）

## 首期支持范围

- `.lang`
- `.json`
- `.js`

- 键值对文本提取
- 翻译项浏览与编辑

## 为什么要做这个项目？

因为很多玩家真正需要的，不只是“一个会翻译的脚本”，而是一个：

- 能打开项目
- 能查看文件
- 能管理翻译项
- 能批量编辑
- 能导出结果
- 能长期维护

的**图形化翻译工作区工具**。

Mod Lingo 想做的，就是把 Minecraft 模组 / Addon 翻译这件事，从零散文件处理，变成一个更清晰、更稳定、更接近日常开发工具体验的流程。
